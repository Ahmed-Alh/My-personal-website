A basic guide to Selfhosting. Published on: 1/04/2020, edited on 7/12/2020

In the age of privacy invasions and data breaches, it's now more important than ever to store all of your data locally instead of having your data in some company's data farm. A great way to do this is to make your own server at home to store all of your data in it, so that you can access it sort of like a network attached storage (NAS). This guide is going to try to explain how you can start selfhosting.

Why should you Selfhost?

There are alot of reasons as to why you should selfhost, like:
  • Total control over your data and what you can store on your server.
  • More private than using a service by a company like Google. (i.e no one can snoop in on your data and analyze it)
  • inherently more secure than using an online namebrand service. As big services are targets for hackers, while your small home server would be a way smaller target.
  • You don't need internet to access your data, since you'll have it all stored in your home.
  • Cheaper than using a paid service or a Virtual private server (VPS), as you don't have to pay monthy fees (except for your electricity bill) or subscribe to a service to access your data. And while you're gonna pay more initially for a computer than using a VPS, it will be a bigger bang for your buck than if you bought a Vps with the same configurations.
Moreover, you can selfhost your data on most, if not any computer. Even a $35 Raspberry pi or an old Laptop will suffice.

How you can selfhost

"How can I start selfhosting?" you might ask. Well, you'll first need the following:
  1. A Computer with a gigabit ethernet port.
  2. An Internet connection, Obviously
  3. An ethernet jack. (While a wireless home server is possible, it is highly prefered to use a wired internet connection as your home server will be used to move data back and forth, and that might saturate your wifi network, leaving it very slow.)
  4. A *Nix operating system (cause Windows Server is a meme.)
  5. Storage, such as a hard dive (an external one can be used, but they tend to be less reliable than standalone hard drives)
  6. knowledge of networking and computers will be helpful.

Step 1: Pick an operating system.

without an OS, you just have a metal box of capacitors and PCBs. Picking an operating system is the first step in making a home server.
I'm going to use Gnu/linux here, but Bsd-based OS's can work as well. There are many distros to choose from. Personally, I'd pick one of the following:
  • OpenMediaVault (OMV), a simple to use operating system built for NASes and home servers with a webui.
  • Debian, it's very stable, but default Debian has some pretty outdated packages.
  • Fedora Server and CentOS, They are both based on RHEL and they offer more up to date and stable packages. From what I know, CentOS is more stable and better for server use than Fedora Server
  • Arch linux, if you want a cutting edge server (and bragging rights).
All Gnu/Linux distributions can work well for server use (with tweaking and optimizing of course) but there are specific things you need to look out for in a distribution, like how frequent are the packages upgraded, how stable is the distribution itself, etc.

Step 2: Install it on your Computer.

This step is pretty easy (for most distros at least, looking at you Arch Linux), just get a flash drive, download the ISO, flash it on the flash drive using Rufus for windows ( BalenaEtcher if you're a normie.) or DD for *nix or macOS, complete the installation, then Bob's your uncle! You're half of the way there. Now you just need to install a couple of things that are essential for a home server.

Step 3: finalize the installation and set up some other things

Now for the final part; setting up and installing some things for turning your newly made linux machine into an actual home server. now this depends on what you want your server to do. for example, if you want to have a Network attached storage where you can store your data and move that data more easily between devices, then you have two options:
  • You can setup a samba share on your server, For Basic file sharing,
  • Or you can set up Nextcloud to have a Google drive or a Dropbox equivalent hub with a document editor and plugins to fit your needs.
  • If you want to have a locally hosted netflix, you can use Plex or Jellyfin to have your own at home netflix.
You might also want to access your server over the internet, so if you're planning to do that then make sure to keep everything secure with SSL and make your passwords long and complicated.

Step 4: profit?

with all of that done, you should have a fully functioning home server made to suit your needs. Of course, there are other things that you're going to need to take care of, but that's beyond the scope of this blog. thanks for reading, and I'll see you in the next one.
